Q1 (a) Draw a flowchart to find the sum of all numbers between 1 to 100 that are divisible by 5. Explain the flowchart symbols used. (7 marks)

Solution:

Logic: We need to iterate through numbers from 1 to 100. Inside the loop, we check if the current number is divisible by 5 (i.e., `num % 5 == 0`). If it is, we add it to a `sum` variable.

[Flowchart Structure]

( START ) Oval
|
[ sum = 0, i = 1 ] Rectangle (Process)
|
< i <=100?> Diamond (Decision)
/ \
YES NO ----> [ PRINT sum ] Parallelagram (Input/Output)
/ \ |
< i % 5==0?> ( STOP ) Oval
/ \
YES NO
/ \
[ sum+=i ] \
\ /
\ /
[ i++ ]
|
(Loop Back)
Explanation of Symbols:
  • Oval (Terminator): Used for START and STOP. Indicates the beginning and end of the flow.
  • Parallelogram (Input/Output): Used for displaying the final result ("PRINT sum").
  • Rectangle (Process): Used for calculations and assignments (e.g., `sum = sum + i`, `i = i + 1`).
  • Diamond (Decision): Used for conditions (e.g., `i <= 100`, `i % 5==0`). It has two outgoing paths (Yes/No).
  • Arrows (Flowlines): Connect the symbols and show the direction of execution.
Software Development

Q1 (b) Discuss the significance of testing and debugging in programming. What are the different types of errors encountered during programming? (6 marks)

Solution:

1. Significance of Testing and Debugging

Testing and debugging are critical phases in the software development life cycle (SDLC) to ensure software quality.

  • Testing: The process of executing a program with the intent of finding errors. It verifies that the software meets specific requirements and behaves as expected.
    Goal: To discover bugs.
  • Debugging: The process of locating, analyzing, and fixing the errors found during testing. It involves tracing the code to find the root cause of a failure.
    Goal: To fix bugs.
Topper's Note: Testing proves the "presence" of bugs, but never their "absence". Debugging is the remedial action taken after testing.
2. Types of Errors
  • Syntax Errors (Compile-time Errors):
    Violations of the language's grammar rules. The compiler detects these and prevents execution.
    Example: Missing semicolon (;), unmatched braces {}.
  • Logical Errors (Semantics Errors):
    The program runs but produces incorrect output due to flawed logic. These are the hardest to detect.
    Example: Using area = l + b instead of area = l * b.
  • Runtime Errors:
    Errors that occur during execution, causing the program to crash.
    Example: Division by zero, accessing invalid memory (segmentation fault).
  • Linker Errors:
    Occur when the program cannot link to the required object files or libraries.
    Example: Undefined reference to a function (e.g., writing 'prntf' instead of 'printf' and not including a header).

Algorithm

Q2 (a) Write an algorithm to check whether a given number is prime or not. Also, discuss the characteristics of a good algorithm. (6 marks)

Solution:

Algorithm: Check Prime Number
  1. START
  2. Read integer N.
  3. If N <= 1, Print "Not Prime" and GOTO Step 8.
  4. Initialize counter i = 2.
  5. Repeat Steps 6 while i <= N/2:
  6. If (N % i == 0), Print "Not Prime" and GOTO Step 8.
    7. Else, increment i by 1.
  7. Print "Prime Number".
  8. STOP.
Characteristics of a Good Algorithm:
  • Finiteness: It must terminate after a finite number of steps.
  • Definiteness: Each step must be clear and unambiguous.
  • Input: It should accept zero or more inputs.
  • Output: It should produce at least one output.
  • Effectiveness: Operations must be basic enough to be performed exactly.

1. Data Types in C

C provides a rich set of data types to define the type of data a variable will hold.

Category Types Description
Primary (Primitive) int, float, double, char, void Basic building blocks.
Derived Array, Pointer, Function Derived from fundamental types.
User-Defined Structure, Union, Enum Defined by user for complex data.
2. Rules for Constructing Variables (Identifiers)
  • Characters: Can contain letters (A-Z, a-z), digits (0-9), and underscore (_).
  • Start: Must start with a letter or an underscore. Cannot start with a digit.
    Valid: _count, num1 | Invalid: 1num
  • No Special Symbols: No spaces or symbols like @, #, $, % are allowed.
    Valid: total_sum | Invalid: total sum
  • Reserved Keywords: Cannot use C keywords (int, float, if, while, etc.) as variable names.
  • Case Sensitive: 'Sum' and 'sum' are different variables.

Control Structures

Q3 (a) Discuss various looping statements used in C language (while, do-while, for). Write the differences between them with syntax examples. (6 marks)

Solution:

Loops are used to execute a block of code repeatedly until a specific condition is met.

1. while Loop

Entry-controlled loop. Condition checks before execution.

while (condition) { // code }
2. do-while Loop

Exit-controlled loop. Executes at least once.

do { // code } while (condition);
3. for Loop

Compact loop. Init, condition, and update in one line.

for(init; cond; upd) { // code }
Recursion vs Coding

Q3 (b) Write a C program to display the factorial of a given number using both recursive and non-recursive methods. Explain which approach is more efficient. (7 marks)

Solution:

#include <stdio.h> // Recursive Function long factorialRecursive(int n) { if (n == 0) return 1; return n * factorialRecursive(n - 1); } // Non-Recursive (Iterative) Function long factorialIterative(int n) { long fact = 1; for (int i = 1; i <= n; i++) { fact=fact * i; } return fact; } int main() { int num; printf("Enter a number: "); scanf(" %d", &num); printf("Factorial (Recursive): %ld\n", factorialRecursive(num)); printf("Factorial (Iterative): %ld\n", factorialIterative(num)); return 0; }
Efficiency Analysis: The Iterative (Non-Recursive) approach is generally more efficient in terms of memory. Recursion builds up a call stack for every function call, which can lead to Stack Overflow for large numbers and consumes more memory overhead. Iteration uses a single stack frame.

Type Conversion

Q1 (b) What do you understand by type conversion? Explain implicit and explicit type conversion with examples. (6 marks)

Solution:

Type Conversion: Process of converting one data type into another during program execution.

1. Implicit Conversion (Type Coercion)

Done automatically by the compiler. Smaller types are promoted to larger types (e.g., int to float) to prevent data loss.

int x = 10; char y = 'A'; // ASCII 65 // 'y' is implicitly converted to int (65) before addition int sum = x + y; // result 75
2. Explicit Conversion (Type Casting)

Done manually by the programmer using a cast operator `(type)`. Used when forcing a conversion that might lose data (e.g., float to int).

float a = 10.5; // Explicitly casting float to int int b = (int)a; // b becomes 10 (fractional part lost)

Bit Logic / Looping

Q3 (b) Write a C program to convert a decimal number to binary using loops and conditional statements. Display the binary equivalent. (7 marks)

Solution:

To convert Decimal to Binary, we repeatedly divide the number by 2 and store the remainder. The binary number is the remainders read in reverse order.

#include <stdio.h> int main() { int n, binary[32], i = 0; printf("Enter a decimal number: "); scanf("%d", &n); // Handling 0 case if (n == 0) { printf("Binary: 0\n"); return 0; } // Conversion Logic while (n > 0) { binary[i] = n % 2; // Store remainder (0 or 1) n = n / 2; // Divide by 2 i++; } // Printing in reverse printf("Binary: "); for (int j = i - 1; j >= 0; j--) { printf("%d", binary[j]); } printf("\n"); return 0; }
